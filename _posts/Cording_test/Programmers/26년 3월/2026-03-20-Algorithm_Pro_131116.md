### 📌 문제 설명

다음은 식품의 정보를 담은 `FOOD_PRODUCT` 테이블입니다. `FOOD_PRODUCT` 테이블은 다음과 같으며 `PRODUCT_ID`, `PRODUCT_NAME`, `PRODUCT_CD`, `CATEGORY`, `PRICE`는 식품 ID, 식품 이름, 식품코드, 식품분류, 식품 가격을 의미합니다.

| Column name | Type | Nullable |
| --- | --- | --- |
| PRODUCT_ID | VARCHAR(10) | FALSE |
| PRODUCT_NAME | VARCHAR(50) | FALSE |
| PRODUCT_CD | VARCHAR(10) | TRUE |
| CATEGORY | VARCHAR(10) | TRUE |
| PRICE | NUMBER | TRUE |

<br>

### 📌 문제

`FOOD_PRODUCT` 테이블에서 식품분류별로 가격이 제일 비싼 식품의 분류, 가격, 이름을 조회하는 SQL문을 작성해주세요. 이때 식품분류가 '과자', '국', '김치', '식용유'인 경우만 출력시켜 주시고 결과는 식품 가격을 기준으로 내림차순 정렬해주세요.

<br>

`FOOD_PRODUCT` 테이블이 다음과 같을 때

### 코드

#### ver(1) - 다중 컬럼 IN 서브쿼리 활용

가장 직관적으로 많이 사용하는 방식입니다. `GROUP BY`로 카테고리별 최대 가격을 먼저 구한 뒤, 그 결과와 정확히 일치하는 행을 메인 쿼리에서 찾아냅니다.

```sql
SELECT CATEGORY, PRICE AS MAX_PRICE, PRODUCT_NAME
FROM FOOD_PRODUCT
WHERE (CATEGORY, PRICE) IN (
    SELECT CATEGORY, MAX(PRICE)
    FROM FOOD_PRODUCT
    GROUP BY CATEGORY
    HAVING CATEGORY IN ('과자', '국', '김치', '식용유')
)
ORDER BY MAX_PRICE DESC;
```

#### ver(2) - Window Function (RANK() OVER) 활용

RANK() 윈도우 함수를 사용하여 서브쿼리 내에서 순위를 매긴 후, 1등(가장 비싼)인 데이터만 뽑아내는 아주 깔끔하고 모던한 풀이법입니다.

```sql
WITH RankedProducts AS (
    SELECT CATEGORY, PRICE AS MAX_PRICE, PRODUCT_NAME, 
           RANK() OVER (PARTITION BY CATEGORY ORDER BY PRICE DESC) AS rnk
    FROM FOOD_PRODUCT
    WHERE CATEGORY IN ('과자', '국', '김치', '식용유')
)

SELECT CATEGORY, MAX_PRICE, PRODUCT_NAME
FROM RankedProducts
WHERE rnk = 1
ORDER BY MAX_PRICE DESC;
```

#### ver(3) - 상관 서브쿼리 (Correlated Subquery) 활용 💡

다중 컬럼 IN이나 윈도우 함수를 쓰지 않고, WHERE 절 안에서 메인 쿼리와 서브쿼리를 연결하여 푸는 또 다른 방법입니다. 각 행을 검사할 때마다 해당 카테고리의 최고가를 서브쿼리로 조회하여 비교합니다.

```sql
SELECT CATEGORY, PRICE AS MAX_PRICE, PRODUCT_NAME
FROM FOOD_PRODUCT AS A
WHERE PRICE = (
    SELECT MAX(PRICE)
    FROM FOOD_PRODUCT AS B
    WHERE A.CATEGORY = B.CATEGORY
)
AND CATEGORY IN ('과자', '국', '김치', '식용유')
ORDER BY MAX_PRICE DESC;
```
